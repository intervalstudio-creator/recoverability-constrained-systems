Real Cases

RECOVS is validated through real-world failures and interventions where recoverability was either lost or restored.

Case 1 - Healthcare (intervention)

A bedbound patient received no follow-up for years after diagnosis.

A caregiver applied the recoverability condition:

detected uncertainty
identified absence of records
triggered escalation

The system corrected:

→ patient referred for reassessment

Result: Non-admissible continuation detected and stopped.

Case 2 - Healthcare (failure)

No escalation despite patient deterioration.

System conditions:

no detection
no response
no recovery

Result:

→ continuation occurred under non-admissible conditions
→ irreversible risk increased

Case 3 - Institutional failure

A system was present but no action occurred.

signals existed
response did not occur
enforcement failed

Result:

→ non-admissible continuation
→ system failure despite visibility

Case 4 - NHS transport failure

A documented continuity failure where delay prevented recovery.

time constraint exceeded
coordination failed
recovery path collapsed

Result:

→ continuation should not have occurred
→ system crossed irreversible boundary

What all cases show

Failure does not come from lack of information.

Failure comes from:

inability to act in time
absence of enforcement
loss of recoverability

System rule

If a system cannot:

detect
respond
and recover in time
→ it is already unsafe
→ continuation must not occur
